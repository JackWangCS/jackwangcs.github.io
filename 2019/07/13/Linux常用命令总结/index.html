压缩包管理

tar

tar 命令用于归档（打包，archive）文件，其目的是将多个文件或目录打包成一个tar文件。由于很多压缩算法只能针对单个文件进行压缩，因此打包成tar包以后，会方便进一步的压缩。例如，可以先将目录打包成tar包，然后再通过gzip/bzip进行压缩。

软件安装和管理

不同的系统具有不同的软件管理工具，对于Ubuntu(Debian)，则通常使用apt/apt-get工具；对于MacOS，则使用brew。此外，还有一部分软件通过源码方式进行发布，因此需要手动的进行构建。

对于Debian系统，使用了.deb文件对应用程序进行打包，从而方便应用的分发。.deb文件包含了应用程序运行所需要的各种文件以及应用的元数据，如应用的依赖等。而dpkg则是用于安装应用程序的工具，但是dpkg只能安装应用，对.deb文件的元数据内容进行查看等，不能自动的安装应用的依赖，因为dpkg并没有访问repositories的功能。

apt/apt-get

apt(Advanced Packaging Tool)是apt-get的封装版本，具有更友好的用户接口，差别。apt在安装应用时，会自动解析应用的依赖，然后根据目前已经安装的应用确定即将要安装的依赖应用。用法：

从源码编译安装

不同的源码采用了不同的构建工具，这里主要介绍Python库、CMake等。

Python库

源码中通常需要包含setup.py文件，用于构建python应用。

CMake/Make

通常的步骤：

  1. 解压：tar -xzvf archive-name.tar.gz，然后cd到解压的目录
  2. 配置：./configure
  3. 构建：make
  4. 安装：sudo make install

文件和目录的管理

任务执行

默认情况下，在命令行终端中启动时，就在前台启动该新进程，并且命令行终端 则为新启动的进程的父进程。如果需要在后台执行该任务，可以在该进程后面添加&，表示在后台启动。

通过下面的命令来对启动的后台任务进行管理：

  • jobs显示后台任务
  • fg pid将任务调到前台执行
  • bg pid让一个任务继续在后台执行

需要注意的如果命令行终端退出，那么系统会向它的所有子进程都发送TERM信号，使得普通进程都退出。如果需要在命令行终端退出后依然继续执行该任务，可以使用nohup。nohup会对每个一个进程发送一个SIGHUP信号，使得在nohup命令执行的进程忽略终端关闭的TERM信号。例子：

服务管理

crontab

crontab用于提交和管理用户需要周期性执行的任务。系统中有一个crond进程，该进程会定期检查和执行crontab文件中需要定期执行的任务。

crontab的文件格式：

  • 第1列分钟0～59
  • 第2列小时0～23（0表示子夜）
  • 第3列日1～31
  • 第4列月1～12
  • 第5列星期0～7（0和7表示星期天）
