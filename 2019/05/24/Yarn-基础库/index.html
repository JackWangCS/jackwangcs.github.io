YARN的基础库主要包括：

  • Protocol Buffers，用于RPC通信中，RPC中所有参数采用Protocol Buffers进行序列化和反序列化。
  • Apache Avro，Hadoop生态系统中的RPC框架，最初的设计目 的是解决YARN RPC兼容性和扩展性差等问题，目前用于记录MapReduce应用程序日志。
  • RPC库，仍采用MRv1中的RPC库，但是默认的Writable序列化方法由Protocol Buffers替代。
  • 服务库和事件库，YARN将所有的对象服务化，以便统一管理，服务之间采用事件机制进行通信，不再使用MRv1中基于函数调用的方式。
  • 状态机库，YARN采用有限状态机描述一些对象的状态以及状态之间的转移，从而简化代码结构。

服务库

对于生命周期较长的对象，YARN采用了基于服务的对象管理模型，特点：

  • 将每个被服务化的对象分为四个状态
    • NOTINITED，被创建
    • INITED，已初始化
    • STARTED，已启动
    • STOPPED，已停止
  • 可以对任何服务状态的变化定义监听器执行一些动作
  • 可以对任意服务进行组合，以便统一管理

事件库

YARN采用了基于事件驱动的并发模型，从而提高系统的整体性能。YARN将各种处理逻辑抽象成事件（Event）、事件调度器（AsyncDispatcher）和对应的事件处理器（Handler），并且将每个事件的处理过程分割成多个用状态机表示的步骤。

核心是中央的异步事件调度器AsyncDispatcher，它维护了注册的事件与事件处理器，并根据接收的事件类型驱动服务的运行。AsyncDispatcher会在单个线程eventHandlingThread线程中通过handlerInstance来处理位于事件队列eventQueue中的事件。

状态机库

在YARN中，状态采用的是确定有限状态机（DFM）每个状态转换由一个四元组表示<preState, postState, event, hook>，并定义了三种状态转换方式：

  • preState : postState : event = 1 : 1 : 1

  • preState : postState : event = 1 : N : 1，注意只是有多个可能的后继状态，后继状态由Hook函数确定，但是最终只会转移到一个后继状态。

  • preState : postState : event = 1 : 1 : N
