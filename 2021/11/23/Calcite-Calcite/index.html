Calcite Overview

Parse and SQL Tree

SqlNode

SqlNode表示一个SQL解析树。SqlNode由SqlParser通过解析SQL语句创建得到，SqlNode因此会持有相关SqlParser解析的位置信息：

因此，解析后的SqlNode可以利用SqlValidator对SqlNode进行验证，结合Catalog等信息保证SqlNode的语义正确性。

SqlNode的子类主要有：

  • SqlLiteral，字面量相关：SqlNumericLiteral、SqlIntervalLiteral、SqlAbstractStringLiteral、SqlAbstractDateTimeLiteral
  • SqlDynamicParam
  • SqlIdenetifier
  • SqlCall：包含DML、DDL以及其他子句等
    • DML：SqlSelect、SqlDelete、SqlUpdate、SqlInsert、SqlMerge、SqlJoin、SqlOrderBy
    • DDL：SqlDdl、SqlCreate、SqlDrop
    • Subquery和Window：SqlWith、SqlAtler、SqlWindow
    • 其他：SqlCase、SqlTableRef、SqlHint、SqlPivot、SqlExplain
  • SqlNodeList
  • SqlDataTypeSepc
  • SqlIntervalQualifier

SqlOperator

SqlOperator表示SqlNode中SqlCall的类型，每个Operator有其对应的operands。

SqlKind

SqlKind枚举了SqlNode的可能类型，可以通过SqlNode.isA()来判断SQL的类型。通过阅读SqlKind，可以对SQL支持的语法和类型有一定理解。所有的SQL语句可以大概分为三类（TOP_LEVEL）：

  • QUERY

  • DML

  • DDL

除了此外，SqlKind还定义包含Expression、SET、Functions等其他类型。

Algebra

RelNode

RelNode的定义如下：

A RelNode is a relational expresion

表示一个关系表达式。关系表达式描述如何处理数据，因此名字通常为动词。核心的RelNode包括：TableScan, TableModify, Values, Project, Filter, Aggregate, Join, Sort, Union, Intersect, Minus, Window and Match等。

有两种方式创建一个RelNode：

  • 通过SqlToRelConverter将由SqlNode组成的SQL语法解析树转换而来
  • 通过RelBuilder构造一个RelNode

当RelNode要进行实现时，Calcite将会创建一个RelImplementor来管理该过程。每个可实现的RelNode都包含一个RelTraitSet来描述它的物理属性（physical attributes）。RelTraitSet总是包含一个名为Convention的Trait，用于描述述该RelNode如何将数据传递到其使用的RelNode。对于每一种calling-convention，有一个与之对应的RelNode的子接口。例如，EnumerableRel有EnumerableConvention与之对应，并由EnumerableRelImplementor实现。在实现RelNode时，RelNode必须实现它的calling-convetions的接口。

RelNode接口继承自RelOptNode, RelOptNode是Planner中的节点表示。提供了AbstractRelNode的抽象实现，具体的RelNode需继承AbstractRelNode。

RelTrait

RelTrait用于表示RelNode的Trait（特征，物理特征），如RelNode的convention（用于描述如何接收和传输底层数据），数据的是否有序RelCollation，数据的物理分布RelDistribution等特征。

RelTrait的定义由RelTraitDef来描述，多个RelTrait可能会共用一个RelTraitDef。

RelTraitSet

RelTraitSet表示一个有序的RelTrait集合，内部了维护一个RelTrait构成的数组：

RelTraitSet除了提供对单个RelTrait的获取，替换和转换等操作以外，还以整个RelTrait集合的视角提供了：

  • 判断RelTraitSet是否满足另外一个RelTraitSet

  • 判断两个RelTraitSet是否匹配

Convention

Convention是一种特殊的RelTrait，用于说明RelNode转换成为PhysicalNode时采用的Convention。Calling Convention可以理解为一个特定数据引擎协议（数据源），拥有相同Convention的RelNode可以直接操作。比如JDBC的JDBCRel都有JdbcConvention，Calcite的内建的Enumerable算子EnumerableXXX都有EnumerableConvention，表示数据是Java变量。

Convention之间可以通过ConverterRule进行转换。ConventionTraitDef中将这些转换信息存储在ConversionData。ConversionData包含了一个以Convention为顶点，ConverterRule为边的转换图conversionGraph：

ConventionTraitDef中，定义了一个conversionCache，用于缓存ConversionData：

ConventionTraitDef中，通过registerConverterRule()和deregisterConverterRule()两种方法来添加ConverterRule，从而对底层的ConversionData进行维护。

当对一个RelNode的节点的Convention进行转换时，记该RelNode的Convention为fromConvention，则从conversionGraph寻找以fromConvention到toConvention的一条以ConverterRule构造的路径，然后通过应用该路径上的ConverterRule对RelNode依次进行转换，得到转换后的RelNode。在此过程中：

  1. 转换过程中的中间节点RelNode(i)的RelOptCost可能未知或者无穷大，此时可以根据allowInfiniteCostConverters来判断路径是否有效
  2. 对于一对(fromConvention, toConvention)，可能存在多条ConverterRule都可以使用，选择第一条ConverterRule

RexNode

RexNode表示一个行表达式（row expression），每一个行表达式的类型（type）都必须是已知的。一些常见的RexNode包括：RexLiteral、RexVariable，RexCall（调用算子）等，所有RexNode的子类都是不可变的。

RexNode由RelBuilder创建。

SqlKind的是一个表示SqlNode类型的枚举类型，当将SqlNode转换为RexNode时，保留了SqlKind。

RexLiteral

RexLiteral表示一个常量表达式，通过RexBuilder.makeLiteral()方法创建。一个RexLiteral定义如下：

RexLiteral可以是表示多种类型的常量：NULL、BOOLEAN、DECIMAL、DATE、TIME、INTERVAL、CHAR、BINARY和SYSMBOL等，提供了：

  • 用于构造RexLiteral的toLiteral(RelDataType type, Comparable<?> value)、fromJdbcString(RelDataType type, SqlTypeName typeName, @PolyNull String literal)静态方法

  • 用于比较值与类型是否匹配的静态方法：

  • 用于将值进行转换的public <T> @Nullable T getValueAs(Class<T> clazz)方法

RexVariable

RexVariable主要有三种实现：

  • RexSlot：子类为RexInputRef和RexLocalRef。RexInputRef是对输入关系表达式的一个field的引用。
    • RexInputRef支持多个输入，比如输入的#0: EMP(EMPNO, ENAME, DEPTNO) #1: DEPT(DEPTNO AS DEPTNO2, DNAME)，那么field #4则为DEPTNO2。
    • RexLocalRef用于表示局部变量，RexLocalRef持有一个自增的基于CopyOnWriteList的SelfPopulatingList用于保存局部变量的名字，局部变量的命令规则为$t + n（n为自增的序号）
  • RexDynamicParam：表示行表达式中的一个动态参数，用?n表示
  • RexCorrelVariable：主要用于Nested Loop Join中，用于表示来自Join的另一个关系表达式的行输入变量

RexCall

RexCall表示由一个算子（SqlOperator）构成的表达式（可以理解为UDF）。算子可以有零个或者多个行表达式作为为输入（operands）。

对于RexCall，有下面几个子类：

  • RexSubQuery：表示标量表达式，比如IN，EXISTS的等子句。RexSubQuery持有一个RelNode来表示相关的关系表达式，并且根据不同的子句类型，拥有不同的SqlOperator：

  • RexWinAggCall：表示Window聚集函数

  • RexOver：用于在Window上调用一个聚集函数

  • RexMRAggCall：主要用于实现MATCH_RECOGNIZE RexNode。
