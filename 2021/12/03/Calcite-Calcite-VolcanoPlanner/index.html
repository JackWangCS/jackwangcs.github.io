VolcanoPlanner是一种基于代价的优化器，是Calcite中默认的优化器类型。Volcano在优化的过程中使用了动态规划算法，整体的过程如下：

  1. 将成功匹配的优化规则加入到匹配队列中
  2. 从匹配队列中依次取出匹配的优化规则，并应用优化规则对执行计划进行转换
  3. 迭代上述过程直到达到固定的最大迭代次数，或者执行计划的最优代价不再改变

基本概念

RelSet

RelSet是指一组语义等价的关系表达式集合：

RelSet is an equivalence-set of expressions that is, a set of expressions which have identical semantics.
We are generally interested in using the expression which has the lowest cost.
All of the expressions in an RelSet have the same calling convention.

RelSet的核心成员变量：

在优化的时候，可能存在多个等价的RelSet，此时，这些RelSet按照树形结构组织，并通过Union-Find的方式快速识别两个RelSet是否属于同一个等价集合中。如果两个RelSet是等价的，还会合并（merge）两个RelSet，详细的合并过程可以查看VolcanoPlanner.merge(RelSet, RelSet)的实现。

RelSubset

RelSubset表示具有相同的RelTrait（物理属性）的RelSet的子集：

Subset of an equivalence class where all relational expressions have the same physical properties.

RelSubset相对于RelSet，强调了物理属性相同，这里的物理属性是指RelNode的所有Traits。RelSubset继承了AbstractRelNode，因此也是一种RelNode。RelSubset会持有RelSet的引用，并不保存实际的等价的关系表达式，是一个逻辑概念。

VolcanoRuleMatch

VolcanoRuleMatch表示一个优化规则与目标的关系表达式匹配：

A match of a rule to a particular set of target relational expressions, frozen in time.

VolcanoRuleMatch继承自VolcanoRuleCall：

VolcanoRuleCall则继承自RelOptRuleCall抽象类。

VolcanoPlanner的定义如下，下面只包含了核心的数据结构：

流程

总体

优化关系表达式R的算法流程：

  1. 注册R
  2. 对于可以应用的RelOptRules，创建Rule-Calls（VolcanoRuleCall），并加入到RuleQueue中
  3. 依次从RuleQueue中取出VolcanoRuleCall，并应用RelOptRule
  4. 重复上述步骤

  2. 如果一个subset的父亲不止一个，那么它可以触发任意父亲的subset的规则匹配

  4. 触发优化规则可能会使得多个subset合并

实现

用户代码主要是通过Program来调用VolcanoPlanner，调用过程为：

主要的调用流程：

sequenceDiagram
    participant user As User Code
    participant VolcanoPlanner
    participant RuleDriver
    participant RuleQueue
    participant RuleMatchCall
    participant RelOptRule
    user->>VolcanoPlanner: setRoot()
    VolcanoPlanner-->>VolcanoPlanner: registerImpl()
    user->>VolcanoPlanner: findBestExp()
    loop (RuleQueue not empty) or (max iterations reached)
    VolcanoPlanner->>RuleDriver: driver()
    RuleDriver->>RuleQueue: popMatch()
    RuleQueue->>RuleMatchCall: onMatch()
    RuleMatchCall->>RelOptRule: onMatch()
    RelOptRule->>RuleMatchCall: transformTo(RelNode)
    RuleMatchCall->>VolcanoPlanner: ensureRegistered(RelNode)
    VolcanoPlanner-->>VolcanoPlanner: registerImpl()
    RuleDriver->>VolcanoPlanner: canonize()
  end
    VolcanoPlanner->>user: RelSubset.buildCheapestPlan()

addRule()

利用addRule()方法向Planner中添加优化规则。在添加的过程中，会将每条优化规则的操作数添加到classOperands中。对于ConverterRule，如果是感兴趣的Trait时，还会调用RelTraitDef.registerConverterRule()进行注册，该注册会向Trait的conversionGraph中添加对应的边：

与addRule()方法相对应的是removeRule()，removeRule()是addRule()的逆过程：从classOperands中删除属于该优化规则的操作数、如果是ConverterRule，调用RelTraitDef.deregisterConverterRule()取消注册。

setRoot()

setRoot()用于设置需要优化的关系表达式，在setRoot()中会执行：

  1. 注册Metadata RelNodes，关于Metadata相关的信息，可以参考：Relational Expression Metadata
  2. 调用registerImpl()注册新的关系表达式，并触发规则匹配
  3. 初始化rootConvention并调用ensureRootConverters()来保证Root的RelNode有对应的ConverterRules

registerImpl()是setRoot()的核心，整个过程较为复杂。主要有两个目的：

  1. 通过RelNode和RelSet构建该RelNode所属的RelSubset，如果一个RelNode的RelSubset已经构建，直接返回即可。
  2. 对于新注册的RelNode，调用fireRules(RelNode)触发规则匹配。

fireRules()

fireRules(RelNode)会触发与输入RelNode相匹配的规则。通过classOperands快速查找与该RelNode相关的RelOptRuleOeprand，并通过RelOptRuleOeprand.match()方法进行匹配。

如果RelNode匹配了某一条优化规则，则创建VolcanoRuleCall和DeferringRuleCall，调用其match()方法：

DeferringRuleCall中，将该次规则匹配成功封装为VolcanoRuleMatch对象，并加入到RuleQueue队列中。

findBestExp()

findBestExp()用于获得以root RelNode的具有最小代价的关系表达式。findBestExp()通过RuleDriver不断地从RuleQueue中取出匹配的优化规则，并对RelNode按照优化规则进行转换，得到新的等价集合以及对应RelNode的最小代价（RelSubset表示）。之后，通过RelSubset.buildCheapestPlan()获得最小代价的执行计划：

RuleDriver用于根据给定的算法引用优化规则。VolcanoPlanner有两种RuleDriver：

  1. IterativeRuleDriver：重复执行多次，每条优化规则可能被执行多次。IterativeRuleDriver将会迭代RuleQueue中所有的VolcanoRuleMatch，直到IterativeRuleQueue为空。
  2. TopDownRuleDriver：以自顶向下地方式应用优化规则，从而保证应用优化规则的顺序。TopDownRuleDriver主要用于对空间进行剪枝，以及优化规则匹配互斥检查等。

VolcanoPlanner使用配置CalciteSystemProperty.TOPDOWN_OPT来决定使用何种RuleDriver，默认情况下使用IterativeRuleDriver。
