Flink的故障恢复主要通过Restart Strategies和Failover Strategies来实现。Restart Strategies用于决定是否以及何时失败/受影响的Tasks可以进行重启，Failover Strategies则是决定哪些Tasks需要重启以恢复整个job。

Restart Strategies

可以通过flink-conf.yaml或者动态参数来配置Restart Strategy：

Key Default Type Description
restart-strategy (none) String Defines the restart strategy to use in case of job failures. If checkpointing is disabled, the default value is none. If checkpointing is enabled, the default value is fixed-delay with Integer.MAX_VALUE restart attempts and ‘1 s‘ delay.

此外，在代码中也可以设置：

支持的Restart Strategies：

  • none, off, disable: No restart strategy

  • fixeddelay, fixed-delay: Fixed delay restart strategy

    Key Default Type Description
    restart-strategy.fixed-delay.attempts 1 Integer The number of times that Flink retries the execution before the job is declared as failed if restart-strategy has been set to fixed-delay.
    restart-strategy.fixed-delay.delay 1 s Duration Delay between two consecutive restart attempts if restart-strategy has been set to fixed-delay. Delaying the retries can be helpful when the program interacts with external systems where for example connections or pending transactions should reach a timeout before re-execution is attempted. It can be specified using notation: “1 min”, “20 s”

  • failurerate, failure-rate: Failure rate restart strategy

    Key Default Type Description
    restart-strategy.failure-rate.delay 1 s Duration Delay between two consecutive restart attempts if restart-strategy has been set to failure-rate. It can be specified using notation: “1 min”, “20 s”
    restart-strategy.failure-rate.failure-rate-interval 1 min Duration Time interval for measuring failure rate if restart-strategy has been set to failure-rate. It can be specified using notation: “1 min”, “20 s”
    restart-strategy.failure-rate.max-failures-per-interval 1 Integer Maximum number of restarts in given time interval before failing a job if restart-strategy has been set to failure-rate.

  • exponentialdelay, exponential-delay: Exponential delay restart strategy

    Key Default Type Description
    restart-strategy.exponential-delay.backoff-multiplier 2.0 Double Backoff value is multiplied by this value after every failure,until max backoff is reached if restart-strategy has been set to exponential-delay.
    restart-strategy.exponential-delay.initial-backoff 1 s Duration Starting duration between restarts if restart-strategy has been set to exponential-delay. It can be specified using notation: “1 min”, “20 s”
    restart-strategy.exponential-delay.jitter-factor 0.1 Double Jitter specified as a portion of the backoff if restart-strategy has been set to exponential-delay. It represents how large random value will be added or subtracted to the backoff. Useful when you want to avoid restarting multiple jobs at the same time.
    restart-strategy.exponential-delay.max-backoff 5 min Duration The highest possible duration between restarts if restart-strategy has been set to exponential-delay. It can be specified using notation: “1 min”, “20 s”
    restart-strategy.exponential-delay.reset-backoff-threshold 1 h Duration Threshold when the backoff is reset to its initial value if restart-strategy has been set to exponential-delay. It specifies how long the job must be running without failure to reset the exponentially increasing backoff to its initial value. It can be specified using notation: “1 min”, “20 s”

Implementations

位于flink-core中的RestartStrategies类用于从Flink Configuration中读取Restart Strategy相关的配置，并创建各个策略对应的RestartStrategyConfiguration子类。该RestartStrategyConfiguration将会被保存到ExecutionConfig，用于在Job运行时，创建对应的RestartBackoffTimeStrategy。

在运行时，JobManager中Scheduler会通过RestartBackoffTimeStrategyFactoryLoader的createRestartBackoffTimeStrategyFactory()静态方法，根据RestartStrategyConfiguration创建对应的RestartBackoffTimeStrategy.Factory，并根据Factory创建具体的RestartBackoffTimeStrategy。

Failover Strategies

目前，Flink支持的Failover Strategy有两种，通过jobmanager.execution.failover-strategy配置：

Restart all tasks in the job to recover from a task failure full
Restart pipelined region region

RestartPipelinedRegionFailoverStrategy

RestartPipelinedRegionFailoverStrategy会根据失败的executionVertexId计算需要重启的PipelinedRegions。规则如下：

  1. 包含失败任务的Pipelined Region总是会重启，用failedRegion表示
  2. 如果需要重启的Pipelined Region的输入（Result Partition）不可用，那么包含用于生成该结果的Task的Pipelined Region也需要重启
  3. 如果一个Pipelined Region需要重启，那么它所有的下游消费者也需要重启

RestartPipelinedRegionFailoverStrategy创建时，需要传入ResultPartitionAvailabilityChecker，用于检测规则2中的输入Result Partition的可用性情况。
